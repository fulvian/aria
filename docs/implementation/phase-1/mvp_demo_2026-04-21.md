## 1. Demo Overview

This document records the live demonstration of the 5 MVP use cases from blueprint §1.4.

### MVP Use Cases

1. **Ricerca tematica** — Thematic research with report generation
2. **Triage email** — Daily email triage via scheduled task
3. **Gestione calendario** — Calendar event creation with HITL
4. **Analisi documento** — PDF document analysis and summarization
5. **Conversazione persistente** — Persistent conversation with memory recall

## 2. Use Case 1: Ricerca Tematica

### Objective
User asks ARIA to research a topic and generate a report saved to Drive.

### Command
```
"ARIA, fai una ricerca approfondita su vector DBs 2026"
```

### Expected Flow
1. Search-Agent orchestrates Tavily/Brave/Firecrawl
2. Results distilled into a report
3. Report saved to Google Drive
4. Telegram notification with Drive link

## 3. Use Case 2: Triage Email

### Objective
Scheduled daily task reads inbox, classifies emails, generates digest.

### Command
```
# Scheduled at 08:00
aria schedule run daily-email-triage
```

### Expected Flow
1. `gmail.search` for unread last 24h
2. `gmail.read` for each message
3. Classify: urgent/actionable/info/newsletter
4. Save digest to memory with tag `email_digest`
5. Telegram digest to user

## 4. Use Case 3: Gestione Calendario

### Objective
User asks to schedule a meeting; ARIA proposes slots, user approves via HITL.

### Command
```
"ARIA, pianifica call 30min con Mario la prossima settimana"
```

### Expected Flow
1. Parse request (attendee, duration, date range)
2. `calendar.list` for free/busy next 7-14 days
3. Propose 3 candidate slots via HITL inline keyboard
4. User selects slot via Telegram
5. `calendar.create` event
6. Save event ID to memory

## 5. Use Case 4: Analisi Documento

### Objective
User shares PDF; ARIA extracts content and provides key points summary.

### Command
```
"ARIA, leggi <PDF attached> e dimmi punti chiave"
```

### Expected Flow
1. Receive PDF attachment from Telegram
2. `pdf-extract` skill processes with PyMuPDF
3. Content ingested as episodic memory
4. Key points extracted and summarized
5. Telegram reply with summary + memory reference

## 6. Use Case 5: Conversazione Persistente

### Objective
New Telegram session; user asks about past conversation; ARIA recalls from memory.

### Command
```
"Ricorda cosa abbiamo deciso su aria-ops server?"
```

### Expected Flow
1. `aria-memory/recall` with semantic query
2. Retrieve episodic entries about "aria-ops server"
3. Synthesize context from previous sessions
4. Provide answer with memory reference
